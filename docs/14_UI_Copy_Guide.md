\# Vendly UI Copy Guide

Version: 1.0  
Status: Production Ready

\---

\# 1\. Purpose

This document defines the standard copywriting guidelines and reusable  
text used throughout Vendly.

Goals:  
\- Maintain a professional tone  
\- Reduce ambiguity  
\- Improve consistency  
\- Enhance user trust  
\- Simplify localization in the future

\---

\# 2\. Voice & Tone

Vendly should sound:

\- Professional  
\- Clear  
\- Helpful  
\- Confident  
\- Concise  
\- Action-oriented

Avoid:  
\- Technical jargon  
\- Blameful language  
\- Excessive marketing language

\# 5\. Authentication Messages

\#\# Success

"Welcome back\!"

"Your account has been created successfully."

"You have logged in successfully."

\#\# Errors

"Invalid email or password."

"Please enter a valid email address."

"This account does not exist."

"Your session has expired. Please log in again."

\---

\# 6\. Profile Messages

Success:

"Your profile has been updated successfully."

Errors:

"Please complete all required fields."

"The uploaded file could not be processed."

\---

\# 7\. Verification Messages

Pending:

"Your account is currently under review."

Approved:

"Your account has been verified."

Rejected:

"Your verification request was not approved. Please review the comments and resubmit."

\---

\# 8\. Opportunity Messages

"No opportunities match your current filters."

Success:

"Your application has been submitted successfully."

"We couldn't submit your application. Please try again."

\# 9\. Query Messages

\# 12\. Loading Text

\- "Loading..."  
\- "Fetching data..."  
\- "Preparing dashboard..."  
\- "Uploading file..."  
\- "Please wait..."

Prefer skeleton loaders over text when possible.

\---

\# 13\. Confirmation Dialogs

Delete:

"Are you sure you want to continue? This action cannot be undone."

Approve:

"Approve this submission?"

Reject:

"Reject this submission? Please provide a reason."

Logout:

"Are you sure you want to log out?"

\---

\# 14\. Validation Messages

"This field is required."

"Enter a valid email address."

Mobile:

"Enter a valid mobile number."

Website:

"Enter a valid website URL."

File:

"Unsupported file type."

"The selected file exceeds the maximum allowed size."

\# 15\. Generic Error Messages

"Something went wrong. Please try again."

"Please check your internet connection and try again."

\# 16\. Success Toasts

\- "Saved successfully."  
\- "Changes published."  
\- "Application submitted."  
\- "Verification updated."  
\- "Settings updated."

\---

\# 17\. Accessibility Copy

Buttons should use descriptive labels:

Good:  
"Download BOQ"

Bad:  
"Click Here"

Links should describe their destination.

\---

\# 18\. Date & Time Format

Preferred:

DD MMM YYYY

Example:

18 Jun 2026

Time:

02:30 PM IST

\---

\# 19\. Currency Format

Use localized formatting.

Example:

₹ 12,50,000

Avoid inconsistent separators.

\---

\# 20\. Copy Principles

\- Be concise.  
\- Be respectful.  
\- Use active voice.  
\- Explain errors clearly.  
\- Tell users what to do next.  
\- Avoid unnecessary technical language.
